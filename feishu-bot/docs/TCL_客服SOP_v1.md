## 3. 退换货政策

### 3.1 标准退货 (客户发起)
- **时限**: 收货后 **30 天**内
- **手续费**: TV 类 **15%** restocking fee + 客户承担退货运费；手机/平板 **5%** restocking fee
- **流程**:
  1. 确认订单日期在 30 天内
  2. 告知客户 restocking fee 及退货运费政策
  3. 客户确认后 → 联系仓库生成 return label
  4. 收到退货 → Shopify 执行退款 (扣除 restocking fee)
  5. 更新 Google Sheet 追踪表

### 3.2 产品缺陷退货
- **费用**: 免费退货，不收任何费用
- **流程**:
  1. 要求客户提供缺陷照片/视频
  2. 确认缺陷属实 → 联系仓库生成 **prepaid** return label
  3. 提供换货或全额退款选项
  4. 硬件级问题 (屏幕、主板等) → 引导客户联系 **TCL Call Center** 走保修

### 3.3 货损退货
- **流程**: 同产品缺陷流程
- 必须获取货损照片存档

### 3.4 发货后取消
- **费用**: Restocking fee + 往返运费由客户承担
- **说明**: 已出库包裹无法拦截时，按退货流程处理

### 3.5 断货 (ZNT 缺货)
- **方案 A**: 告知客户预计等待 **15 个工作日** + 赠送 **5% 折扣券**
- **方案 B**: 全额退款
- 由客户自行选择方案

## 4. 价格保护 (Price Match)

| 条件 | 规则 |
|------|------|
| 时限 | 购买后 **30 天**内 |
| 适用渠道 | **仅限 DTC 订单** (naec.store 下单) |
| 不适用 | Amazon、BestBuy、其他渠道购买 |
| 退差方式 | 计算差额 → Shopify 部分退款 |
| 税差 | 价格差包含税额差异 (已统一处理) |
| 操作权限 | 客服可直接执行，无需审批 |

## 5. 物流问题处理

### 5.1 标准订单 (小件快递)
- 客户查询物流 → 先在 Shopify/领星 查 tracking
- 物流异常 → 让客户先联系承运商
- 承运商要求 sender 联系 → 仓库代为联系承运商
- 仓库联系后仍无法解决 → 全额退款

### 5.2 LTL 大件订单 (TV 等)
- LTL 订单可能无自动 tracking number
- 需要手动在领星/Shopify 录入 tracking
- 大件退货需协调 LTL pickup，流程较长
- 保持客户沟通，每 3 天更新一次状态

### 5.3 Return Label 跟进
- **注意**: 仓库 (Coco/ZNT) 生成 return label 可能需要 **3-10+ 天**
- 客服需主动催促，超过 5 天无 label → 在群内 @ 小金跟进
- 已知痛点：仓库响应慢，需要多次催促

## 6. 特殊场景处理

### 6.1 B2B / 批量询价
- **不可自行报价** → 转交 Mingyi 处理
- 参考价: NXTPAPER 14 批量价 $300/台 (仅供内部参考，以 Mingyi 最终报价为准)
- B2B 客户邮件 → Mingyi 审阅后发送

### 6.2 欺诈/重复订单
- Shopify Flow 自动检测: PO Box 地址、重复订单、同一客户多单
- 触发 Flow 标记 → 暂停发货 → 人工审核
- 确认欺诈 → 取消订单 + 退款
- 确认正常 → 解除 hold，正常发货

### 6.3 保修/线下购买咨询
- DTC 不处理保修 → 引导客户联系 **TCL Call Center**
- 线下渠道 (BestBuy/Walmart) 购买 → 引导回购买渠道

### 6.4 产品咨询
- **禁止** 问客户"你收到了什么配件" → 自行查 Shopify 产品页面确认包含内容
- 技术问题 (连接、设置、故障排除) → 引导至 TCL Call Center

### 6.5 安装服务 (Mounting Service)
- 目前推进中，目标 5 月关闭方案确认
- 涉及第三方安装合作 → 具体流程待 Mingyi 确认后更新

## 7. 邮件沟通规范

### 7.1 发件身份
- **统一使用**: naec.store@tcl.com
- **禁止**: 使用个人邮箱、其他公司签名

### 7.2 签名格式
```
Best regards,
TCL Customer Service Team
naec.store@tcl.com
```
- **禁止使用** 外部公司签名 (如 Cell Digital, N2M 等) → 客户会混淆

### 7.3 语气与格式
- 简洁、专业、不啰嗦
- **禁止 AI 腔调** (过度客套、重复致歉、冗长开头)
- 直接说明解决方案 → 客户操作步骤 → 结尾
- B2B 邮件 → 必须 Mingyi 审阅后发送

### 7.4 回复时效
- 常规工单: **24 小时内**首次回复
- 紧急 (退款/缺陷): 当日回复
- B2B 询价: 转 Mingyi 后 **48 小时内**回复

### 8.1 Google Sheet 更新
- **每个工单** 必须录入内部追踪表
- 字段: 订单号、客户姓名、问题类型、状态、处理人、更新日期

- 所有退款、订单编辑需在 Shopify 留 note
- Note 格式: `[日期] [操作类型] [原因简述]`

## 9. 升级路径

```
Level 1: 客服专员 — 标准退换货、价格保护、物流查询、产品咨询引导
    ↓ 超出政策/不确定
Level 2: Mingyi — 政策例外、B2B 报价、仓库升级、特殊退款审批
    ↓ 重大损失/战略决策
Level 3: Barron — 重大客诉、合作决策、制度变更
```

**升级触发条件**:
- 退款金额 > $500
- 客户要求法律/媒体介入
- B2B 批量订单
- 产品召回/批量质量问题
- 政策模糊地带无先例
