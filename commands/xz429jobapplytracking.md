## PHASE 2 — Cross-Reference and Status Computation

For each application in unified list:

**Step A: Age calculation**
```
age_days = (today - date_applied) in integer days
```

**Step B: Email match**
Fuzzy-match `company` from ledger against `company` from email signals:
- Normalize: lowercase, remove Inc/LLC/AI suffix, strip spaces
- Match if: normalized strings share ≥ 5 chars or one contains the other
- On match: record `{ matched_signal, matched_date }`

**Step C: Status inference**
Apply SKILL.md signal rules. Precedence (highest wins, never downgrade):
```
offer > interview > screening > email_reply > submitted
```

If no email match:
- age ≥ 14 days + status = submitted → `ghosted`
- age 7-13 days + status = submitted → `follow_up_due`
- age < 7 days → keep current status

Special statuses (never auto-change): `rejected`, `fill_failed`, `manual_review`, `skipped*`

**Step D: Priority assignment**
```
🔴 = screening | interview | offer | manual_review | fill_failed | follow_up_due (age 7-13)
🟡 = email_reply | submitted (age 1-6)
⚪ = ghosted | rejected | skipped* | submitted (age 0)
```

## AUTO-RECOVERY RULES (MOP v4.5)

| Failure | Recovery |
|---------|----------|
| Gmail MCP wrong account / timeout | Skip 1A, proceed ledger-only, note in report |
| Ledger file missing | Create empty file with header, skip that platform's data |
| Date parse error on ledger row | Skip that row, log count of skipped rows in report footer |
| JobTracker dir missing | `mkdir -p` via Bash, continue |
| Edit fails on ledger update | Skip ledger write-back, note in report, don't block |
| Report write fails | Print full report to console instead |
| Zero applications found | Write empty report with note, don't error |

**Never stop. Never ask. Always produce output.**
