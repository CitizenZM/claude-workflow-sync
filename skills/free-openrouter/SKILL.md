# free-openrouter — Zero-Cost LLM Router

**Hard rule:** This skill MUST NEVER call a paid model. Every model ID it sends ends in `:free` or is on the verified free-only allowlist. If all free options fail, raise — do not fall back to paid.

**API key location:** `~/.claude/credentials.json` → `ai_platforms.openrouter.api_key`
**Helper:** `~/.claude/skills/free-openrouter/or_free.py`
**Endpoint:** `https://openrouter.ai/api/v1/chat/completions`

## When to use this skill

Trigger when any workflow, agent, or script needs:
- A chat completion / LLM call where cost matters
- Bulk enrichment, classification, summarization, extraction
- A second-opinion model that won't burn Anthropic credits
- Vision / OCR on images at zero cost
- Long-context reading (Qwen3 Next gets 262K free)

Do **not** use this skill for:
- Tasks that need Claude-specific behavior (artifacts, tool use, computer use)
- Anything where output quality is more important than cost
- Latency-sensitive paths (free tier latency is variable, 1–8s typical)

## Safety: how this skill prevents accidental paid calls

1. The router whitelist (`or_free.py:JOBS`) only contains models with `:free` suffix or verified free-only IDs (`openrouter/free`).
2. `chat()` raises `PaidModelError` if you pass a model not in the allowlist or without `:free`.
3. `refresh.sh` rebuilds `free-models.json` by querying the OpenRouter `/models` endpoint and filtering `pricing.prompt == 0 AND pricing.completion == 0` — never trust the model ID string alone.
4. If a free model is rate-limited (429), the helper retries the next model in the same job category, never escalating to paid.

## Rate limits (verified from OpenRouter docs)

- **20 requests/min per :free model** — circuit breaker handles via 60s cooldown
- **1,000 requests/day** total across all `:free` models (if ≥$10 lifetime credit purchased) — currently active on this account
- **50 req/day** if <$10 lifetime purchased
- **Negative balance → 402** even on free models. Skill auto-checks with `check_balance=True` arg
- See `BUDGET.md` for daily token throughput, parallelism limits, and workload examples

```bash
python or_free.py --usage   # live API state + balance
python or_free.py --stats   # local daily count + cooldowns + per-model usage
python or_free.py --reset   # clear local state if needed
```

## Optimization (LRU rotation + circuit breaker)

`chat_for(job, prompt)` does NOT hit models in fixed primary→fallback order. It sorts the job's model list by usage count (least-used first), so daily load spreads across all working models. This maximizes the 20-RPM-per-model ceiling.

On 429 → 60s cooldown for that model, route to next.
On 5xx → 30s cooldown, route to next.
On 402 → fail fast (raise OpenRouterError) — paid escalation forbidden.

State persists in `state.json` next to the skill, survives across processes.
